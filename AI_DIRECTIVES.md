2. CONSTRAINT-FIRST BUILD PROTOCOL (THE SKILL)
This is the operational workflow for every build, every directive, every line of code. No execution begins until Phase 1 is complete.
Phase 1 — MD Review (MANDATORY FIRST STEP)
Before any code, design, or content work begins:
Load the AI Directives (this file) — universal non-negotiables.
Load the Project MD — build-specific constraints (see Section 3 template).
Load the Session Handoff MD — current session state and active threads.
Acknowledge constraints. Flag any conflict between them before proceeding.
Phase 2 — Execution (LOCKED INSIDE THE GUARDRAILS)
All work happens only within the constraint envelope.
Optimize freely on anything not locked (layout micro-decisions, naming, animation timing, etc.).
If a directive requires breaking a constraint, stop and surface it. Do not silently work around it.
Do not request permission for low-risk tasks already scoped by the MDs. Execute.
Phase 3 — Self-Validation (PRE-HANDOFF CHECK)
Before delivery, the executing AI confirms:
Brand identity intact (colors, fonts, founder assets, trilingual structure).
Functional scope honored (no scope creep, no unrequested features).
Security boundaries respected (no exposed credentials, no compliance breaks).
Existing structural code preserved.
Phase 4 — Audit & Smoke Test (BIG JIM + ANTIGRAVITY)
Big Jim audits the code and architecture against technical standards.
Antigravity runs the smoke test on the deployed/built artifact.
Failures route back to Phase 2 with the specific violation flagged. No full rebuilds for cosmetic deltas.
Phase 5 — Handoff
Update the Session Handoff MD with what changed, what's open, and what the next session needs.

3. PROJECT MD TEMPLATE (THE INSTANCE)
Every new build, repo, or project gets its own Project MD using this structure. Drop this into the project root before work begins.
```markdown
# PROJECT: [Project Name]
**Build Type:** [Website / PWA / App / Campaign / Document]
**Status:** [Active / Paused / Archived]

## LOCKED CONSTRAINTS
### Brand Identity
- Colors: [Hex codes — locked]
- Fonts: [Header / body — locked]
- Required assets: [Founder photo, logo, etc.]

### Functional Scope
- What this build DOES: [bullet list]
- What this build DOES NOT do: [bullet list — explicit exclusions]

### Security & Legal
- Credentials: [What handles them — never expose]
- Compliance: [HIPAA / GDPR / PCI / N/A]
- Data boundaries: [What can/cannot leave the system]

## OPEN FOR OPTIMIZATION
- [Anything the AI can iterate on freely]

## ACTIVE DIRECTIVES
- [Current sprint goals or specific orders from CEO]
```
Rule: If it's not in `LOCKED CONSTRAINTS`, it's fair game for the AI to optimize. If it IS locked, it does not move without CEO order.
